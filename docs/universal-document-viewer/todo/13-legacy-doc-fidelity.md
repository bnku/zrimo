# Задача 13. Legacy DOC structured conversion

**Статус:** 🟠 В работе: stock-parser отклонён, project-owned parser утверждён

## Цель

Перестать представлять plain-text reconstruction как рендер исходного DOC и
создать permissive browser-side pipeline, сохраняющий структуру paragraphs,
runs, sections и tables.

## Источники требований

- [`../01-fidelity-corrective-roadmap.md`](../01-fidelity-corrective-roadmap.md)
- [`./03-office-pipeline.md`](./03-office-pipeline.md)

## Зависимости

- Задача 09 завершена.

## Что входит в задачу

- Немедленный truthful degradation contract.
- Новый bounded Word Binary parser с переиспользованием permissive CFB, IR и
  DOCX writer из `office_oxide`, но без его эвристического DOC converter.
- Formatting/table/section parsing и faithful DOCX serialization.
- DOC-specific fidelity/security corpus и differential tests.

## Что НЕ делаем в этой задаче

- Не подключаем GPL/AGPL/LGPL/MPL runtime converter.
- Не используем server/desktop binary fallback.
- Не угадываем headings, tables или page breaks по plain text.
- Не объявляем весь DOC spec реализованным по одному representative file.

## Основные задачи по слоям

### Feature

- Удалить heuristic title/heading reconstruction из faithful path и выдавать
  явный warning/error для text-only fallback.
- Реализовать отдельный workspace crate `legacy-doc`: использовать публичный
  `office_oxide::cfb` только как bounded container reader, самостоятельно
  разбирать Word Binary structures и строить `office_oxide::ir::DocumentIR`.
- Расширить parser/IR mapping: styles and inheritance, fonts, paragraph
  alignment/spacing/indents, lists, sections/columns, headers/footers, table
  rows/cells/widths/borders/merges и embedded images.
- Сериализовать только доказанные properties в DOCX; unknown features дают
  structured warning, но не fabricated content. Устранить duplicated title.
- Отдельно переоценить XLS/PPT legacy paths: DOC fix не считается автоматическим
  доказательством их fidelity.

### Этапы реализации

1. [`13a-doc-binary-foundation.md`](./13a-doc-binary-foundation.md) — bounded
   FIB/CLX reader, CP↔FC mapping, stories и собственный structural IR boundary.
2. [`13b-doc-formatting-sections.md`](./13b-doc-formatting-sections.md) — STSH,
   PAPX/CHPX FKP, `sprm`, paragraph/run styles, sections и page geometry.
3. [`13c-doc-tables-media-serialization.md`](./13c-doc-tables-media-serialization.md)
   — table grid/merges/nesting, images, headers/footers и доказанная DOCX
   serialization без эвристик.
4. [`13d-doc-browser-qualification.md`](./13d-doc-browser-qualification.md) —
   WASM worker integration, public API enablement, corpus, fuzz, visual and pack
   gates.

### Tests

- Unit tests на каждый binary structure/sprm с маленькими generated byte-level
  fixtures и bounds/error cases.
- Integration corpus: paragraphs/styles, Cyrillic+Latin+CJK, code pages, tables,
  merged/nested cells, page sizes, columns, lists, images, headers/footers и
  corrupted/encrypted CFBF.
- Structural oracle сравнивает page/section count, table grid, cell text/order,
  styles и geometry; plain-text equality не является достаточным gate.
- Visual differential against independently rendered licensed references с
  tolerance; fabricated headings/duplicate titles — hard failure.
- WASM fuzz/resource/time/cancellation tests для PLCF/FKP/sprm/CFBF bounds.

### Docs

- До `go` опубликовать spike decision record с scope, unsupported structures,
  license и estimate.
- После реализации обновить legacy capability matrix и warnings.
- Документировать provenance committed fixtures, не раскрывая private inputs.

## Критерии готовности

- DOC output не содержит элементов, не подтверждённых source formatting.
- Basic и nested tables, paragraph/run styles, sections и page geometry проходят
  structural/visual gates.
- Backend остаётся browser-side, permissive и bounded на untrusted input.
- Release blocker остаётся до завершения 13a–13d; fail-closed нельзя заменить
  частичным plain-text render между этапами.

## Результат spike и принятое продолжение

Pinned `office_oxide@0.1.6` доказанно ограничен FIB/CLX plain-text extraction и
images: STSH, PAPX/CHPX FKP, sections и Word table grid отсутствуют. Текущий IR
эвристически создаёт headings и дублирует первый как section title, поэтому его
DOC bytes-out path запрещён в TypeScript adapter и Rust binding. Публичный
runtime возвращает `fidelity-unsupported` с capability details; отдельный
`extractLegacyPlainText` остаётся только диагностическим API. XLS/PPT продолжают
квалифицироваться независимо.

`No-go` относится к stock DOC converter `office_oxide@0.1.6`, а не к формату DOC.
Так как DOC остаётся обязательным форматом продукта, продолжение утверждено как
project-owned permissive parser. До завершения всех четырёх этапов публичный
runtime продолжает fail closed.
